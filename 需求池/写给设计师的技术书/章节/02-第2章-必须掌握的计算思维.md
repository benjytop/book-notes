# 第 2 章 必须掌握的计算思维

## 章节定位

给出设计师看技术的「思维工具」——计算思维四要素。后续章节都基于这套思维工具展开。

## 核心内容

### 2.1 计算思维四要素

1. **抽象**:从众多事物中抽取出共同的、本质性的特征(命名、分类、关注点分离)
2. **分解**:把大问题拆成可管理的小问题(分而治之,直到「可以编程解决」)
3. **模式识别**:在不同数据 / 场景中找到共性(设计规范 = 成功模式的抽象)
4. **算法**:逐步解决问题的步骤(交互流程 = 算法)

### 2.2 设计流程中的计算思维

作者把斯坦福设计学院的设计思维五步映射到计算思维:

| 设计思维 | 计算思维应用 |
|---|---|
| 同理心 | 用户体验地图 = 分解 + 抽象;场景建模 = 分解 + 模式识别 |
| 定义问题 | 用户画像 = 分解 + 模式识别 + 抽象;5W1H = 分解 + 抽象 |
| 创意构思 | 情绪板 = 模式识别 + 抽象;卡片分类法 = 模式识别 |
| 原型制作 | 算法思维贯穿(动效、3D 渲染、AIGC 都依赖算法) |
| 测试 | 眼动追踪 + 点击热图 = 模式识别;漏斗模型 = 分解 + 量化 |

### 2.3 计算思维对设计师的重要性

- 即使不提「抽象、分解、模式识别」这些名词,它们已经渗透到设计过程
- 「算法」可能在原型制作中较多使用,但直接影响设计质量

## 关键洞察

> 计算思维 ≠ 编程。编程是「写汉字、写英文」,计算思维是「构思文章的组织和内容」。掌握计算思维能更快学会新编程语言。
