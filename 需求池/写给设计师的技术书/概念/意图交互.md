# 意图交互(Intent-based Interaction)

## 命题

> **作者主张**:基于意图的交互设计是 AI 时代的关键模式——用户表达「想要做什么」,系统理解意图并执行,而不是按按钮触发固定流程。苹果对智能交互的核心理解就是「意图驱动」。

## 来源 / 出处

- 章节:第 5.3 节「基于意图的交互设计」
- 重点:5.3.1「苹果对于智能交互的理解」

## 核心机制

### 意图交互的三要素

```text
用户输入(自然语言 / 语音 / 文本)
   ↓
意图识别(系统理解用户「想要做什么」)
   ↓
执行(可能是 App 跳转 / 操作 / 信息呈现)
```

### 与传统 GUI 的对比

| 维度 | 传统 GUI | 意图交互 |
|---|---|---|
| 用户输入 | 点击按钮 / 选择菜单 | 自然语言 / 语音 / 文本 |
| 系统响应 | 固定流程 | 动态理解 + 执行 |
| 反馈 | 视觉变化 | 文字 / 语音 / 视觉 |
| 学习成本 | 用户学操作 | 用户表达意图 |

### 设计挑战:意图的设计和泛化

- **意图设计**:列出「用户可能想做的所有事」,为每个意图设计执行路径
- **意图泛化**:用户表达「打开客厅的灯」,系统也要能理解「把客厅灯打开」「开灯,客厅」等不同说法
- **意图失败**:系统不能理解时,进入 [[准确率与兜底|兜底流程]]

## 使用步骤

1. **列出核心意图**:从用户高频需求出发,列出 10–20 个核心意图
2. **收集表达方式**:每个意图至少 50 种不同说法(口语化、方言、缩写)
3. **设计意图路径**:每个意图对应一个执行流程
4. **设计兜底**:意图识别失败的兜底——参见 [[准确率与兜底]]
5. **持续迭代**:从真实用户表达中提取新意图与新说法

## 适合场景

- **语音助手**:Siri、小爱同学、车载语音
- **智能客服**:用户提问 → 系统理解 → 给出答案 / 转人工
- **AIGC 工具**:用户描述需求 → 生成内容
- **智能家居**:"把客厅灯调暗一点" → 找到灯设备 → 调整亮度

## 限制与反例

- **意图边界**:不是所有操作都适合意图交互——复杂配置、多步骤操作仍需要 GUI。
- **意图冲突**:用户表达含糊时(「打开它」),系统不知道「它」指什么。
- **意图误识别**:「打开窗户」误识别为「打开网页」→ 严重后果。关键操作必须确认。
- **意图 ≠ AI**:意图可以靠规则实现(正则表达式 + 关键词),不一定需要 AI。
