# C1-C12 易读性约束 (Readability Constraints)

12 个强约束 (Hard Constraints) + 5 个柔性建议 (Soft Suggestions)。

## C1: 章节文件排序 (Hard)

**强约束**:
- 章节文件命名: `NN-{中文章名}.md`, NN 为两位阿拉伯数字 (01-99)
- 部目录命名: `1-部名/`, `2-部名/`, ... (Obsidian 字母数字排序)
- 每部前言: `00-前言-{部名}.md`
- 附录: `52-后记.md`, `53-致谢.md`, `54-附录.md`, `55-译后记.md`, `56-版权.md`

**实现**: `s50_build_vault.py` 在创建章节时自动加 NN- 前缀。

**Obsidian 排序行为**:
- ✅ 稳定: `01-第一章.md` < `02-第二章.md` < `10-第十章.md`
- ❌ 混乱: `第一章.md` < `第八章.md` < `第二章.md` (按 Unicode 而非数值)

## C2: 段落长度 (Hard)

**强约束**: 每个 markdown 段落 ≤ 8 行 (移动端友好)。

**实现**: LLM prompt 明确: "每个段落不超过 8 行, 超过请用空行分段"。

## C3: 标题层级 (Hard)

**强约束**:
- `##` 用于 vault 内主段 (e.g., `## 书籍信息`, `## 豆瓣数据`)
- `###` 用于子段 (e.g., `### 内容简介`, `### 作者简介`)
- `####` 可选细分子段
- 禁用 `#####+`

**实现**: LLM prompt 明确: "标题最多到 #### 级别"。

## C4: 引文格式 (Hard)

**强约束**:
- 原文引用必须用 `> blockquote`
- 禁止 LLM 改写原文 (除非明确标注"改写:")
- 短评精选 (豆瓣评论) 原文搬运, 不改
- 关键引用标注章节位置

**示例**:
```markdown
> "就像指南针总是指向北方一样, 男人怪罪的手指头总是指向女人。"
> —— [[娜娜]] (第一章)
```

## C5: 表格列数 (Hard)

**强约束**: 表格列数 ≤ 5。

**超 5 列的解决方案**:
- 拆成 2 个表
- 或转成项目列表

## C6: 图片 (Hard)

**强约束**:
- 不嵌入 base64 图片
- 不引用绝对路径 (e.g., `/Users/...`)
- 不引用外部 URL 图片
- 封面图可选放 vault 根目录 (作为图片文件, 不嵌入)

**理由**: 用户明确拒绝嵌入封面图 (版权 + 渲染问题)。

## C7: 段间空行 (Hard)

**强约束**: 每个段落、列表、代码块前后必须有空行。

**实现**: `s50_build_vault.py` 的 normalizer 自动加空行。

## C8: 中文全角标点 (Hard)

**强约束**:
- 中文段落强制全角: `，。！？；：（）「」『』`
- 禁用英文标点混入中文 (e.g., `Hello, world.` 不能在中文段落里出现)
- 例外: 英文专有名词、技术术语、引用、URL

**实现**: LLM prompt 明确: "使用中文全角标点"。

## C9: 数字格式 (Hard)

**强约束**:
- ≥ 10,000 字 → `XX.X 万字` (一位小数)
- < 10,000 字 → 整数
- 数字单位: `21.6 万字`, `428 页`, `8.7 万字`
- 禁止"全文字数 / 正文字数"双口径
- 位置: MOC「书籍信息」段

**实现**: `s50_build_vault.py` 读取 spine 字符数, 按规则格式化。

## C10: 内部链接 (Hard)

**强约束**:
- vault 内交叉引用必须用 `[[wikilink]]`
- 禁用 `[text](file.md)` 形式
- 链接目标必须存在 (broken 被 L1 拒绝)

**关键**: 所有 wikilink 由脚本自动生成, 避免人肉笔误。

**实现**:
```python
# 自动生成
link = f"[[{entity_name}]]"
md_text = md_text.replace("{{玛丽雅姆}}", link)
```

## C11: 日期格式 (Hard)

**强约束**:
- frontmatter: ISO 8601 `2026-07-20`
- 文档内: 阿拉伯数字 + 中文月日 `2026 年 7 月 20 日`
- 禁用英文月名 `Jul 20, 2026`

## C12: 字数规则 (Hard)

**强约束**:
- 字数计算源: EPUB spine **全量字符数** (不去除扉页)
- 数字格式: ≥ 10,000 → `XX.X 万字`, < 10,000 → 整数
- 位置: MOC「书籍信息」段
- 禁止双口径

**实现**:
```python
spine_chars = sum(len(f.read_text()) for f in spine_dir.glob("*.txt"))
if spine_chars >= 10000:
    word_count = f"{spine_chars/10000:.1f} 万字"
else:
    word_count = f"{spine_chars} 字"
```

## 5 个柔性建议 (Soft)

### S1: 表格 vs 列表
- 3 列以内用表格, 3+ 列用列表

### S2: 代码块语言标记
- 代码块用 ` ```python ` 而不是 ` ``` `

### S3: 强调符号
- 中文用 `**bold**` (markdown 通用)

### S4: 链接 alt text
- 链接可加显示文本 `[[玛丽雅姆|玛丽雅姆 (主角 A)]]`

### S5: 段落结构
- 关键论点段先点题, 再展开
- 段落之间可用过渡段
