# 人脸识别(Face Recognition)

## 命题

> **作者主张**:人脸识别不是单一技术,而是「检测 → 追踪 → 识别 → 活体检测」的完整链路。每一步都有失败场景,设计必须考虑兜底。

## 来源 / 出处

- 章节:第 7 章「人脸识别和追踪」

## 核心机制

### 完整链路(5 步)

```text
1. 检测(Face Detection)        ← 找到画面中有没有人脸
2. 追踪(Face Tracking)         ← 持续追踪人脸位置
3. 识别(Face Recognition)      ← 判断「这是谁」
4. 活体检测(Liveness Detection)← 判断「是不是真人而非照片 / 视频」
5. 表情分析(Facial Analysis)   ← 判断「情绪、注意力」
```

### 5 个常见应用领域

| 应用 | 关键能力 | 设计要点 |
|---|---|---|
| 闸机通行 | 检测 + 识别 + 活体 | 1 秒内完成,失败要快速兜底 |
| 考勤打卡 | 检测 + 识别 | 需考虑「代打卡」场景 |
| 身份核验 | 检测 + 识别 + 活体 | 高安全场景需要多因素认证 |
| 注意力检测 | 检测 + 追踪 + 表情 | 配合驾驶或学习场景 |
| 美颜 / 滤镜 | 检测 + 关键点 + 表情 | 用户体验为主,精度要求低 |

## 使用步骤(设计师角度)

1. **识别应用类型**:每个应用对 5 步链路的需求不一样
2. **设计失败兜底**:每一步都可能失败,设计 fallback
3. **环境约束**:光线 / 角度 / 遮挡 / 距离都会影响
4. **隐私设计**:摄像头始终开启,需要物理指示灯 + 隐私声明
5. **儿童 / 老人适配**:识别模型需要覆盖多种脸部特征

## 适合场景

- **闸机 / 门禁**:1v1 比对(人脸 vs 数据库)
- **考勤 / 签到**:1vN 比对(人脸 vs 多人数据库)
- **支付**:1v1 比对 + 活体检测
- **车载 DMS**:检测 + 追踪 + 表情,判断驾驶员疲劳
- **教育**:检测学生注意力

## 限制与反例

- **光线影响**:逆光、极暗光、强逆光 → 准确率显著下降
- **遮挡影响**:口罩、墨镜、刘海、帽子 → 部分特征缺失
- **多胞胎 / 相似**:人脸高度相似时容易误识
- **种族偏差**:训练数据偏美国人 → 其他人种准确率低(作者强调)
- **隐私争议**:人脸数据涉及重大隐私,需谨慎处理
- **「活体检测」不是万能**:高清照片、视频仍可能骗过部分活体检测
