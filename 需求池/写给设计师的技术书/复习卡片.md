# 复习卡片

> 一问一答,适合在通勤 / 排队 / 睡前快速回忆。每个卡片都是原书某个具体知识点的浓缩。

## 思维工具

### 卡片 1:计算思维四要素

**Q**:计算思维的四个公认要素是什么?它们在解决复杂问题时的配合流程是什么?
**A**:抽象、分解、模式识别、算法。流程:①抽象+分解把复杂问题拆小 → ②抽象+模式识别理解问题 → ③模式识别找数据相似性 → ④算法逐步解决。
**来源**:第 2.1 节 → [[计算思维]]

### 卡片 2:计算思维 vs 编程

**Q**:作者为什么强调「计算思维 ≠ 编程」?
**A**:编程是「写汉字、写英文」(掌握和计算机直接交流的语言),计算思维是「构思文章的组织和内容」(具备写出正确程序的思维)。掌握计算思维能更快学会新编程语言,但不必懂代码也能用计算思维。
**来源**:第 2.1 节

### 卡片 3:MVC 三角色

**Q**:MVC 三个角色各关注什么?为什么叫「关注点分离」?
**A**:View(视图)关注显示 / Model(模型)关注数据 / Controller(控制器)关注事件与流程。三者互相独立——模型不依赖视图和控制器,视图无程序逻辑,控制器连接两者。这就是「关注点分离」,让设计师和工程师能各自专注自己的模块。
**来源**:第 3.1 节 → [[MVC架构模式]]

### 卡片 4:原子设计 5 层级

**Q**:Brad Frost 的原子设计 5 个层级是什么?
**A**:原子(无法分解的基本元素)→ 分子(原子集合形成的简单组件)→ 有机体(形成界面离散部分的复杂组件)→ 模板(展示底层内容结构)→ 页面(填入真实内容)。
**来源**:第 2.1.1 节 → [[原子设计]]

## 跨设备

### 卡片 5:8 种屏幕状态

**Q**:动态布局需要考虑的 8 种屏幕状态是什么?
**A**:①手机/折叠屏折叠态 ②Pad/折叠屏展开态 ③折叠屏支架态 ④多窗口态 ⑤小窗态 ⑥桌面态 ⑦Widget ⑧通知。
**来源**:第 4.1.1 节 → [[跨设备动态布局]]

### 卡片 6:苹果设备四象限

**Q**:苹果设计师 Cas Lemmens 在 WWDC 2017 提出的设备四象限是什么?
**A**:按「个人/多人」×「移动/固定」划分。Apple Watch = 最移动 + 个人(健身 App 在这);Apple TV = 最固定 + 多人(没有健身 App)。
**来源**:第 4.2.1 节

## AI

### 卡片 7:训练集 / 验证集 / 测试集

**Q**:三类数据集的类比、占比、作用分别是什么?
**A**:训练集(~80%,平时学的知识)、验证集(~10%,课后习题)、测试集(~10%,期末考试)。验证集用于调参,测试集用于评估泛化能力。
**来源**:第 5.1.1 节 → [[神经网络]]

### 卡片 8:数据标注三原则

**Q**:数据标注的三大原则是什么?
**A**:均衡(不同类别数目接近)、纯净(杂质少,特征清晰)、场景多样(角度/距离/背景多样)。
**来源**:第 5.1.1 节 → [[数据标注]]

### 卡片 9:准确率与兜底阈值

**Q**:作者建议的 AI 输出阈值与处理策略是什么?
**A**:99% 直接展示、80–99% 展示 + 提示确认、60–80% 展示备选项、<60% 走兜底流程。
**来源**:第 5.5 节 → [[准确率与兜底]]

### 卡片 10:兜底 4 个层次

**Q**:语音交互失败的 4 种兜底方案是什么?为什么互斥?
**A**:①告知型(抱歉我听不懂,成本低)②搜索型(传给搜索,适合查询)③闲聊型(传给第三方闲聊机器人,成本最高)④Q&A 型(传给产品内问答,适合「怎么打开 X」)。互斥 = 一次失败只能用一种,叠加会让用户困惑。
**来源**:第 5.5 节

## 感知技术

### 卡片 11:姿态 vs 手势识别

**Q**:姿态识别和手势识别的关键差异是什么?
**A**:姿态关键点 17+(全身,远距离 2–5m),手势关键点 21+(单手,近距离 0.5–2m)。姿态用于健身 / 游戏 / DMS,手势用于智能音箱 / 车载 / AR/VR。
**来源**:第 6.3、6.4 节 → [[姿态识别]] [[手势识别]]

### 卡片 12:人脸识别 5 步链路

**Q**:人脸识别的完整链路是什么?
**A**:①检测(找到有没有人脸)→ ②追踪(持续追踪位置)→ ③识别(判断「这是谁」)→ ④活体检测(判断「是不是真人」)→ ⑤表情分析(情绪、注意力)。每一步都可能失败,都要设计 fallback。
**来源**:第 7.2 节 → [[人脸识别]]

### 卡片 13:眼动追踪「注意力 ≠ 意图」

**Q**:为什么眼动追踪的「注意力 ≠ 意图」?应该如何弥补?
**A**:用户「看向某处」不等于「想点击某处」。必须配合其他输入(眨眼、按钮、手势)才能形成完整交互。眼动是「预演」,其他输入是「确认」。
**来源**:第 8.5 节 → [[眼动追踪]]

## 设备互联

### 卡片 14:RFID vs NFC

**Q**:RFID 和 NFC 的关键差异?
**A**:RFID 距离几米到数十米、标准多、1v1+1vN、安全性较低;NFC 距离 <10cm、标准简单、仅 1v1、安全性较高。NFC 由 RFID 演变而来,主要用于消费电子(门禁/公交/支付)。
**来源**:第 9.1.1 节 → [[RFID与NFC]]

## 未来方向

### 卡片 15:6 个未来方向的一句话瓶颈

**Q**:智能座舱、VR、AR、数字人、空间交互、元宇宙各自的一句话瓶颈是什么?
**A**:智能座舱 = 算力 2–3 代落后;VR = 20ms 延迟制约;AR = 配准误差难以完全控制;数字人 = 建模 vs 渲染平衡;空间交互 = 传感器精度上限;元宇宙 = 全面技术缺口。
**来源**:第 10 章 → [[主题-未来设计的不确定性]]
