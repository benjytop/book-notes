# 眼动追踪(Eye Tracking)

## 命题

> **作者主张**:眼动追踪让用户的「目光」成为输入,可以判断注意力、辅助 GUI 操作、用于研究和无障碍场景。核心限制是摄像头位置和用户配合度。

## 来源 / 出处

- 章节:第 8 章「眼动追踪」

## 核心机制

### 眼动追踪的两大原理

1. **红外反射法**:红外 LED 照射眼睛,摄像头捕捉角膜反射点 + 瞳孔位置 → 计算注视点
2. **可见光法**:普通摄像头 + 视觉算法识别瞳孔和虹膜 → 计算注视点(精度低)

### 关键技术指标

| 指标 | 含义 | 设计含义 |
|---|---|---|
| 精度(Accuracy) | 注视点偏差,通常 <0.5° | 决定 GUI 操作能否落地 |
| 采样率(Sampling Rate) | 每秒采样次数,通常 60–120Hz | 决定交互流畅度 |
| 头动范围(Head Movement) | 头部可移动范围,通常 ±25° | 决定用户是否需要「保持头部稳定」 |
| 校准时间(Calibration) | 个性化校准所需时间 | 影响首次使用体验 |

### 4 类典型应用

| 应用 | 关键能力 | 设计要点 |
|---|---|---|
| 科学研究(可用性测试) | 注视点 + 热图 | 离线分析为主 |
| 注意力检测(车载) | 实时注视方向 | 配合 DMS,疲劳 / 分心检测 |
| 辅助交互(渐冻症患者) | 注视触发操作 | 高容错,长延迟可接受 |
| 消费产品(VR 头显) | 注视点渲染 + 操作 | 配合头动使用 |

## 使用步骤(设计师角度)

1. **明确应用目的**:研究?驾驶?无障碍?不同目的对精度要求不同
2. **设计校准流程**:首次使用必须校准,设计简洁的 3–5 点校准
3. **设计反馈**:用户「看向哪」要有视觉反馈(高亮 / 光标)
4. **设计组合输入**:眼动 + 手势 / 语音往往比单独眼动更稳定
5. **环境约束**:光线 / 眼镜 / 美瞳 / 瞳孔颜色都会影响

## 适合场景

- **可用性测试**:分析用户在界面上的注意力分布
- **辅助技术**:渐冻症、ALS 患者用目光操作电脑
- **车载 DMS**:检测驾驶员是否「看了路」
- **VR 头显注视点渲染**:用户注视方向高分辨率,周边低分辨率(节省算力)
- **广告效果评估**:分析广告的注意力吸引力

## 限制与反例

- **硬件成本高**:高精度眼动仪价格仍贵,Tobii 等品牌数千美元
- **用户配合度**:必须保持「头部不动 + 睁开眼」,长时间使用疲劳
- **戴眼镜问题**:眼镜反光会干扰红外反射
- **环境光线**:强光 / 极暗光都会影响
- **注意力 ≠ 意图**:用户「看向某处」不等于「想点击某处」——必须配合其他输入(眨眼、按钮)
