# 主题:传感器与 AI 驱动的体验设计

## 命题

> **作者主张**:从姿态手势到人脸眼动,每一种传感器都对应一种「新型输入设备」,设计师必须了解它的精度、延迟、环境约束。AI 让这些输入从「确定性」变为「概率性」,必须用「准确率 + 兜底」框架处理。

## 主要论据

### 1. 传感器四件套(第 6–9 章)

| 传感器 | 典型应用 | 关键约束 |
|---|---|---|
| 摄像头(姿态 / 手势) | 健身、车载、AR/VR | 光线、遮挡、距离、延迟 |
| 摄像头(人脸) | 闸机、考勤、支付、DMS | 光线、遮挡、种族偏差、活体检测 |
| 摄像头 / 红外(眼动) | 可用性测试、车载、VR | 头部稳定、戴眼镜、校准 |
| RFID / NFC / UWB(感知) | 支付、配对、资产追踪 | 距离、金属干扰、隐私 |

### 2. AI 时代的「准确率 + 兜底」框架

任何 AI 功能的输出都有概率分布,设计师需要:

```text
高概率(>99%)→ 直接展示
中概率(80–99%)→ 展示 + 提示确认
低概率(60–80%)→ 展示备选项
极低概率(<60%)→ 走兜底(告知 / 搜索 / 闲聊 / Q&A)
```

### 3. 设计工具箱的「四件套」

作者在每个传感器章节末尾都给出:

1. **设计注意事项**:该传感器在什么场景下不能用
2. **设计拆解**:一个真实案例的完整拆解
3. **设计工具箱**:可用于设计的工具列表
4. **未来展望**:该方向的技术瓶颈

## 关键洞察

- **「新型输入设备」视角**:每个传感器 = 一个新型输入设备(类似键盘 / 鼠标 / 触摸屏),有它能做的和不能做的。
- **「准确率 + 兜底」是通用框架**:适用所有 AI 功能,不止于语音助手。
- **「兜底方案不是万能话术」**:「抱歉,我听不懂」如果高频出现反而是减分。
- **「隐私设计不可忽视」**:摄像头始终开启 → 必须有物理指示灯 + 隐私声明。

## 设计含义

- **不要把传感器当作「黑魔法」**:理解每个传感器的精度、延迟、约束,才能设计可落地的体验
- **每个 AI 功能都有失败场景**:为每个失败场景设计 fallback
- **训练数据决定 AI 能力上限**:评估 AI 产品时问「训练数据覆盖了哪些人群」
- **多模融合是趋势**:单一传感器不可靠,组合多种输入更稳定(眼动 + 眨眼触发、手势 + 语音确认)
