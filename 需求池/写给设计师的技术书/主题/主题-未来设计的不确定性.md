# 主题:未来设计的不确定性

## 命题

> **作者主张**:智能座舱、VR、AR、数字人、空间交互、元宇宙等未来方向看起来热闹,背后都有尚未解决的技术难题。设计师要警惕「概念先行」,把「技术瓶颈」作为设计输入而非事后补丁。

## 主要论据(第 10 章 6 个方向)

### 1. 智能座舱(10.1)

- **核心瓶颈**:算力极度缺陷 + 难以遍历完整驾驶任务
- **典型数据**:车载芯片比手机晚 2–3 代,2021 年车型仍用 2015 年算力芯片
- **AR-HUD 6 个未解难题**:贴合精度 / 驾驶员影响 / 动态调整 / 天气适应 / 边界外预警 / 用户信任

### 2. 虚拟现实 / VR(10.2)

- **核心瓶颈**:20 毫秒的延迟制约
- **原因**:头显转动 → 屏幕渲染 → 用户看到画面必须 <20ms,否则晕动症
- **设计含义**:任何涉及头部追踪的交互都要考虑延迟预算

### 3. 增强现实 / AR(10.3)

- **核心瓶颈**:配准误差难以完全控制
- **原因**:现实环境光照 / 物体移动 / 摄像头畸变导致虚拟物体与现实物体的「贴合」有误差
- **设计含义**:AR 内容要设计「容错」,而不是追求「像素级贴合」

### 4. 数字人(10.4)

- **核心瓶颈**:建模和渲染之间的平衡
- **原因**:高精度建模 + 实时渲染 = 极大算力需求
- **设计含义**:数字人的「真实感」与「实时性」是天平两端,需要按场景取舍

### 5. 空间交互(10.5)

- **核心瓶颈**:摄像头、传感器的精度是否支撑设备的交互行为
- **原因**:空间交互依赖多个传感器(深度摄像头、IMU、ToF),精度、范围、延迟三者难以兼得
- **设计含义**:空间交互的体验上限 = 传感器能力上限

### 6. 元宇宙(10.6)

- **核心瓶颈**:实现元宇宙的难度极大
- **原因**:需要算力 + 网络 + 设备 + 内容生态 + 标准协议 + 用户接受度的全面突破
- **作者判断**:元宇宙的实现「在本书出版几年内仍会存在」技术难题

## 关键技术瓶颈对照表

| 方向 | 一句话瓶颈 | 设计应对 |
|---|---|---|
| 智能座舱 | 算力 2–3 代落后 | 不押注旗舰体验,做可降级方案 |
| VR | 20ms 延迟 | 任何交互的延迟预算 <20ms |
| AR | 配准误差 | 设计容错,不追求像素级贴合 |
| 数字人 | 建模 vs 渲染 | 按场景选精度,提前取舍 |
| 空间交互 | 传感器精度 | 体验上限 = 传感器能力上限 |
| 元宇宙 | 全面技术缺口 | 不押注元宇宙做产品核心 |

## 关键洞察

- **「未解决」不等于「不可能」**:这些方向都在快速进步,设计师需要持续跟踪技术现状。
- **「概念先行」是风险**:用户期望 ≠ 技术能力,设计师要为「期望管理」负责。
- **每个方向都有「先落地再升级」路径**:不要追求「完美体验」,先做「可用体验」。
- **「场景适配」是关键**:同一技术在 B 端(可控环境)可能能用,在 C 端(复杂环境)就可能不行。

## 设计含义

- **做产品决策时**:把「技术成熟度」列入风险评估表
- **做设计方案时**:为每个「技术未成熟」的方向准备 fallback
- **做团队沟通时**:用「这个方向 2026 年的技术现状是 X」而非「这个方向很有前景」
- **做用户期望管理**:不要承诺「即将到来的元宇宙体验」,而是「当前可实现的具体功能」
