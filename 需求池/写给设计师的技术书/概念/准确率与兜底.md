# 准确率与兜底

## 命题

> **作者主张**:智能设计最重要的两个因素是「准确率」与「兜底」。AI 输出有 99% / 90% / 80% / 60% 等不同可能性,设计师要设置阈值决定如何处理不同置信度的结果,并为低置信度结果设计兜底方案。

## 来源 / 出处

- 章节:第 5.5 节「智能设计:基于『准确率』和『兜底』的设计」
- 重点:5.5 节是全书的「设计核心方法论」

## 核心机制

### 准确率阈值设计

```text
AI 输出概率
   99% ──→ 阈值以上:直接显示结果(如「这是猫」)
   90% ──→ 阈值边缘:展示但提示用户确认
   80% ──→ 阈值边缘:展示 + 提供备选项
   60% ──→ 阈值以下:走兜底流程
   <60% ──→ 阈值以下:兜底 / 不展示
```

### 兜底的 4 个层次(语音交互案例)

1. **告知型兜底**:用多种说法告知用户「系统听不懂」("抱歉,目前还不能理解你的意思" / "我还在学习中")。优点:成本低、对话自然。缺点:高频出现会让用户觉得产品什么都不懂。

2. **搜索兜底**:把听不懂的语句传给第三方搜索。例如「我想看哈利·波特的视频」→ 用正则提取「视频」「哈利·波特」→ 跳到视频搜索。优点:覆盖广。缺点:用在指令意图上会非常奇怪("打开客厅的灯" → 跳到百度搜索)。

3. **闲聊兜底**:把听不懂的语句传给第三方闲聊机器人。优点:人性化,多轮对话。缺点:成本最高(需要 API),闲聊机器人有自身角色定位。

4. **Q&A 兜底**:把问题传给产品内建的问答系统,适合"怎么打开 ACC" / "天空为什么是蓝的" 这类问题。

四种兜底策略**互斥**,只能选一种作为单次失败的处理路径。

## 使用步骤

1. **识别 AI 输出的不确定性**:每个 AI 功能都问「准确率多少?哪些场景会失败?」
2. **设置阈值**:与产品 / 算法一起确定「可直接展示 / 需确认 / 走兜底」的阈值
3. **设计兜底方案**:为每个失败场景设计兜底——是告知?搜索?闲聊?还是 Q&A?
4. **持续优化**:记录真实失败案例,反哺训练数据 → 提升准确率 → 减少兜底触发

## 适合场景

- **所有 AI 功能设计**:语音、视觉、推荐、生成式内容
- **不确定性的产品**:人脸识别、手势识别、眼动追踪、AI 推荐
- **跨语言 / 跨地区产品**:翻译、跨文化场景下准确率天然低,兜底更关键

## 限制与反例

- **兜底方案 ≠ 万能话术**:「抱歉,我听不懂」如果高频出现,反而是体验减分。
- **兜底策略不能组合**:一次失败只能用一种兜底,多种兜底叠加会让用户困惑。
- **准确率会随场景变化**:实验室 99% 准确率,真实场景可能跌到 70%。
- **阈值是动态的**:对儿童用户可以降低阈值,对专业用户可以提高阈值。
