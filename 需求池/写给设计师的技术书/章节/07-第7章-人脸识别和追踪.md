# 第 7 章 人脸识别和追踪

## 章节定位

摄像头如何「看见」人脸——身份识别、表情分析、注意力检测。

## 核心内容

### 7.1 人脸识别的不同作用

- 闸机通行
- 考勤打卡
- 身份核验(支付、政务)
- 注意力检测(车载 DMS)
- 美颜滤镜
- 表情互动

### 7.2 人脸追踪和识别的技术细节

- 检测 → 追踪 → 识别 → 活体检测 → 表情分析 五步链路
- 检测:找到画面中有没有人脸
- 追踪:持续追踪人脸位置
- 识别:判断「这是谁」
- 活体检测:判断「是不是真人而非照片」
- 表情分析:判断「情绪、注意力」

### 7.3 实时人脸识别的设计注意事项

- 环境因素:光线 / 角度 / 遮挡 / 距离
- 多人场景:追踪优先级选择
- 隐私设计:摄像头始终开启 → 物理指示灯 + 隐私声明
- 失败兜底:识别失败的 fallback(刷脸失败 → 扫码 / 手动输入)

### 7.4 设计拆解:人脸识别闸机

- 1 秒内完成识别
- 失败快速兜底
- 防止「代刷」「照片攻击」

### 7.5 设计拆解:表情设计和互动

- 表情作为输入(微笑拍照、眨眼切歌)
- 表情作为反馈(数字人表情、动画驱动)
- 表情的「夸张化」设计(数字人需要夸张才明显)

### 7.6 设计工具箱

- Face++ / 旷视
- Apple Vision Framework
- Google ML Kit
- OpenCV + DLib

## 关键洞察

> 人脸识别不是单一技术,是「检测 + 追踪 + 识别 + 活体 + 表情」五步链路。每一步都有失败场景,设计必须为每一步考虑兜底。
