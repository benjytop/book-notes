# 原子设计(Atomic Design)

## 命题

> **作者主张**:原子设计是 Brad Frost 提出的界面分解方法,本质上是用计算思维中的「分解 + 命名」把界面拆成可复用的层级单元。

## 来源 / 出处

- 提出者:Brad Frost(原书作者为 Brad Frost 的同名著作)
- 章节:第 2.1.1 节「抽象」
- 转引:作者在解释「抽象」时引入作为典型案例

## 核心机制

5 个层级,从最小到最大:

```text
原子(Atoms)        →  UI 元素,无法进一步分解,界面的基本构建块
   ↓ 组合
分子(Molecules)     →  相对简单的 UI 组件,由原子集合而成
   ↓ 组合
有机体(Organisms)   →  相对复杂的组件,形成界面离散部分
   ↓ 放入布局
模板(Templates)     →  展示设计的底层内容结构(无真实内容)
   ↓ 填入内容
页面(Pages)         →  真实内容 + 模板,用于测试设计系统的弹性
```

### 5 个层级的 Brad Frost 定义

| 层级 | 定义 | 设计举例 |
|---|---|---|
| 原子 | UI 元素,无法进一步分解 | 按钮、输入框、标签、颜色、字体 |
| 分子 | 相对简单的 UI 组件,由原子集合 | 搜索框(图标 + 输入框 + 按钮) |
| 有机体 | 相对复杂的组件,形成界面离散部分 | 导航栏、卡片列表 |
| 模板 | 把组件放进布局,展示内容结构 | 文章详情页布局 |
| 页面 | 真实内容套入模板 | 一篇真实文章的详情页 |

## 使用步骤

1. **建原子库**:从颜色、间距、字号、按钮、输入框等最小元素开始
2. **组合成分子**:把多个原子组合成可复用组件(图标 + 文字 = 标签项)
3. **堆出有机体**:复杂组件(导航栏、卡片列表、评论区)
4. **设计模板**:在真实内容之前,先把结构定下来
5. **填入页面**:用真实内容测试设计系统的弹性(各种长度标题、各种数量图片)

## 适合场景

- **设计系统建设**:原子设计是 Design System 的经典骨架。
- **设计走查**:用「这个设计稿能不能拆到原子层」的视角检视一致性。
- **跨团队沟通**:「这是同一个分子」比「这是同一个搜索框组件」更易理解。

## 限制与反例

- **不是所有界面元素都能 5 级拆分**:有些元素介于「分子」与「有机体」之间,边界判断依赖团队共识。
- **页面与模板的区分有时模糊**:在敏捷迭代中,模板经常直接变成页面。
- **不解决交互问题**:原子设计是「视觉 + 结构」分层,交互逻辑(状态、动效)需要另外的体系(如 Design Tokens、States)。
