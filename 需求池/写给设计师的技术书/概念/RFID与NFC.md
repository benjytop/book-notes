# RFID 与 NFC

## 命题

> **作者主张**:RFID(射频识别)与 NFC(近场通信)是让设备短距离感知彼此的两类硬件方案,适合做身份识别、支付、设备互联。设计核心是「距离 + 场景 + 隐私」。

## 来源 / 出处

- 章节:第 9 章「感知、互联和追踪」,特别是 9.1.1「RFID 和 NFC」

## 核心机制

### RFID vs NFC

| 维度 | RFID | NFC |
|---|---|---|
| 工作距离 | 远(数米到数十米) | 近(<10cm) |
| 频率 | 低频 / 高频 / 超高频 | 13.56MHz |
| 典型应用 | 物流、门禁、资产追踪 | 支付、公交、配对 |
| 供电 | 有源(电池)/ 无源(感应供电) | 无源(感应供电) |
| 安全性 | 较低 | 较高(短距离,难以远距离窃取) |

### RFID 三类标签

1. **无源标签**:靠读写器电磁场供电,价格低,寿命长,适合一次性使用
2. **半有源标签**:有电池但只在被激活时通信,适合物流追踪
3. **有源标签**:自带电池主动广播,适合实时追踪(车辆、贵重物品)

### NFC 的三种工作模式

1. **读卡器模式**:NFC 设备读取 NFC 标签(海报、公交卡)
2. **卡模拟模式**:NFC 设备模拟成卡片(手机变公交卡、门禁卡)
3. **点对点模式**:两个 NFC 设备之间传数据(早期蓝牙配对)

## 使用步骤(设计师角度)

1. **明确使用场景**:远距离追踪 → RFID;近距离支付 / 配对 → NFC
2. **设计距离约束**:RFID 不能「太远」(功率问题);NFC 不能「太近」(要接触或 <10cm)
3. **设计隐私保护**:无源标签无法删除信息,需谨慎写入敏感数据
4. **设计触发反馈**:用户「刷一下」要有明确反馈(震动 + 声音 + 屏幕)
5. **设计失败兜底**:NFC 刷卡失败常见,要有降级方案(二维码 / 手动输入)

## 适合场景

- **移动支付**:Apple Pay、华为 pay、银联闪付
- **公交 / 地铁**:全国交通一卡通、Apple Pay 交通卡
- **门禁 / 闸机**:小区门禁、办公楼门禁
- **资产追踪**:仓库货物、图书馆书籍
- **设备配对**:蓝牙耳机配对、智能家居配网
- **防伪**:商品防伪标签(扫码验证)

## 限制与反例

- **金属 / 液体干扰**:金属和液体会吸收 RFID 信号,导致读取失败
- **多标签碰撞**:多个标签同时进入读写器范围会互相干扰
- **隐私风险**:无源 RFID 标签可被远距离「嗅探」,写入敏感信息后无法删除
- **NFC 速度慢**:NFC 传输速率低(106/212/424 kbps),不适合传大文件
- **标准化问题**:不同厂商 NFC 实现细节差异大,跨平台兼容性挑战
