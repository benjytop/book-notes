# 第 1 章 设计师为什么要懂技术

## 章节定位

全书开篇,从三个角度回答「设计师要不要懂技术」。

## 三条核心理由

### 1.1 行业改变和进步

- 海外懂技术的设计师越来越多
- 互联网公司期待「全链路设计师」(覆盖用户研究 → 设计 → 上线 → 反馈)
- 互联网泡沫、新一代设计师、AI 三股力量同时到来,设计师需要重新定位
- AR/VR、车联网、智能硬件等新赛道出现 → 设计师必须懂技术才能参与

### 1.2 技术和设计之间有着密切的关系

- 技术、设计和创新三者形成闭环
- 技术架构师的核心能力(抽象 + 分解)对设计师有借鉴价值
- 创新需要「技术敏感度」+「设计思维」+「用户洞察」三者的结合

### 1.3 设计过程中需要考虑技术

设计流程中每一步都要考虑技术:

| 设计步骤 | 技术考量 |
|---|---|
| 前期调研 | 技术成熟度调研、竞品技术栈 |
| 用户研究 | 用户的技术使用习惯、跨设备场景 |
| 原型设计 | 哪些交互技术能实现、哪些不能 |
| 用户测试 | 真实设备上的可用性、性能 |
| 评审交付 | 与工程师的架构对话 |

## 关键洞察

> 「懂技术不是要懂代码,而是要懂技术的输入输出、限制、约束」——本章贯穿全文的核心观点。
