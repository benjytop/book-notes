# 无剧透导读

## 一句话定位

面向 UI / 交互 / 体验设计师的「技术通识手册」,从计算思维与架构模式出发,讲清跨设备交互、AI 与传感器(姿态手势、人脸识别、眼动追踪、感知互联)在设计层面的可操作要点,以及智能座舱、VR、AR、数字人、空间交互等未来方向未解决的技术难题。

## 这本书在处理什么问题

设计师的传统训练建立在「屏幕 + GUI + 点击」之上,但 2020 年之后:

- 终端形态扩展到手机、折叠屏、平板、桌面、车机、XR 头显、智能家居;
- 交互方式扩展到语音、姿态、手势、人脸、眼动等「非 GUI 输入」;
- 内容生产端被 AIGC、Web3、生成式 AI 重构。

这三个变化共同要求设计师从「画界面」转向「与技术协同定义可实现的设计」。本书的回答路径是:

```
为什么要懂技术(第1章)
        ↓
用哪种思维理解技术:计算思维(第2章)+ 架构思维(第3章)
        ↓
技术与设计如何结合:跨设备布局(第4章)+ AI(第5章)
        ↓
具体感知技术:姿态手势(第6章)+ 人脸(第7章)+ 眼动(第8章)+ 感知互联(第9章)
        ↓
未来难题:智能座舱、VR、AR、数字人、空间交互、元宇宙(第10章)
```

## 结构总览

- **第 1–4 章**:基础方法论。讲清「为什么」与「用哪种思维看技术」,以及跨设备这一最常见的多端场景。
- **第 5–9 章**:具体技术领域。每章都给「技术原理 + 设计注意事项 + 设计拆解 + 设计工具箱」四件套。
- **第 10 章**:未解难题。把当前看似炫酷的方向(智能座舱 / VR / AR / 数字人 / 空间交互 / 元宇宙)拆给读者看——背后都有尚未解决的技术约束,设计师需要与工程师共同面对。

## 阅读前需要知道的背景

- **不需要编程基础**。作者在前言明确说「不是懂代码才算懂技术」,重点是思维模式与设计层面的输入输出。
- **第 2 章的计算思维**是后续所有章节的「语法」,建议先读。
- **第 3 章的 MVC**只需理解「视图 / 模型 / 控制器」三个角色即可,后续跨设备、AI、感知都复用这套抽象。
- 涉及到具体传感器 / 算法时,**不必死记参数**,关键是知道「在什么场景下用、有什么设计注意事项」。

## 主要人物

本书无虚构人物。作者薛志荣是百度、小鹏、华为人机交互实验室背景的独立设计师,本身就是「懂技术的设计师」的样本。

## 建议阅读方式

- **第一遍**(2–3 小时):读完全书,优先抓章节结论与「设计拆解」「设计注意事项」。
- **第二遍**(主题阅读):按主题横向阅读——例如「跨设备」只看第 4 章;「AI」只看第 5 章;「感知」看 6–9 章。
- **第三遍**(工具箱):把每章末尾的「设计工具箱」汇总成自己未来做设计的 checklist。

## 带着这五个问题阅读

1. **我正在做 / 即将要做的设计,可能涉及哪些技术?** 哪些我可以不懂细节,但必须懂输入输出与限制?
2. **这个技术决策背后,是 GUI 时代的「可视化流程」,还是 AI 时代的「准确率 + 兜底」?**
3. **当终端形态改变(折叠屏、车机、头显),我设计的「视图」会发生什么变化?**
4. **我设计的交互是否依赖某种传感器?传感器的精度、延迟、环境约束会不会让设计「翻车」?**
5. **未来 3 年内,哪些设计方向会因为技术不成熟而推迟?我的产品是否押注了这些方向?**

## 可能的阅读难点

- **术语密度**:第 5–9 章集中出现 AI / CV / HCI 术语,初次接触可能压力大。**应对**:第 2 章的计算思维 + 第 3 章的 MVC 两次铺垫后,再回头读具体技术会顺很多。
- **缺少画面**:本书图文混排但没有配色。**应对**:边读边在草稿上画「视图 / 模型 / 控制器」三块图,把抽象概念落到纸上。
- **行业背景门槛**:车机、XR 章节假设读者了解产业。**应对**:先看前言对 2022 年 XR 政策、AI 浪潮的判断,补背景。

> 说明:本书为非虚构,无剧透概念,本文件本身即为完整导读。
