# 第 8 章 眼动追踪

## 章节定位

用户的「目光」能否成为输入?——可用性研究、注意力检测、辅助交互。

## 核心内容

### 8.1 眼动追踪在不同领域的作用

| 领域 | 关键能力 | 典型产品 |
|---|---|---|
| 学术研究 | 注视点 + 热图 | Tobii Pro |
| 可用性测试 | 注意力分布 | 各种 UX 研究工具 |
| 注意力检测 | 驾驶员是否看路 | 车载 DMS |
| 辅助交互 | 渐冻症患者输入 | Tobii Dynavox |
| VR 注视点渲染 | 节省算力 | Quest Pro |

### 8.2 眼动追踪的技术原理

- **红外反射法**:红外 LED + 摄像头 + 角膜反射 + 瞳孔位置 → 注视点
- **可见光法**:普通摄像头 + 视觉算法 → 注视点(精度低)

#### 关键技术指标

| 指标 | 含义 |
|---|---|
| 精度 | 注视点偏差,通常 <0.5° |
| 采样率 | 每秒采样次数,60–120Hz |
| 头动范围 | 头部可移动范围,通常 ±25° |
| 校准时间 | 个性化校准所需时间 |

### 8.3 设计注意事项

- 摄像头摆放位置:不能被遮挡
- 用户穿着打扮:眼镜反光、墨镜、美瞳、瞳孔颜色都影响
- 光线影响:强光 / 极暗光都降低精度
- 戴眼镜:反射会干扰红外追踪
- 校准流程:首次必须校准,设计 3–5 点简洁流程

### 8.4 设计拆解:Tobii 眼动仪和 GUI 的配合

- Tobii 与 Windows / macOS 的系统级集成
- 注视 = 鼠标悬停的「预演」
- 注视 + 眨眼 / 按键 = 触发
- 渐冻症患者专用交互模式

### 8.5 设计工具箱

- Tobii Pro Spectrum / Fusion
- Tobii Dynavox(辅助技术)
- Pupil Labs(开源)
- Apple Vision Pro 眼动
- Meta Quest Pro 眼动

## 关键洞察

> 眼动追踪的「注意力 ≠ 意图」——用户「看向某处」不等于「想点击某处」。必须配合其他输入(眨眼、按钮)才能形成完整交互。
