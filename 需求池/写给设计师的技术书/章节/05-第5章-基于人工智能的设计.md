# 第 5 章 基于人工智能的设计

## 章节定位

AI 时代的设计核心——用「准确率 + 兜底」框架理解智能设计。本章是全书方法论的「应用层」。

## 核心内容

### 5.1 人工智能基础知识

#### 5.1.1 人工智能模型

- 神经网络 = 多个神经元连接,每条连接有一个参数
- 训练 = 调整参数让损失函数的值变小
- 「有多少人工,就有多少智能」——关键在高质量标注数据
- 数据分为结构化(易)和非结构化(难,占 80%)
- 数据标注三大原则:均衡、纯净、场景多样

#### 5.1.2 基于预测的人工智能

- 神经网络本质是「预测」——输入 → 输出概率分布
- 不同应用有不同的预测目标(分类、回归、生成)
- 「可解释性」是趋势,黑盒模型难以让用户信任

#### 5.1.3 人工智能模型的注意事项

- 数据偏差是最大风险(Flickr 人脸数据偏美国人)
- 泛化能力 = 测试集表现,真实场景 ≠ 验证集表现
- 模型要「见过」才能识别,新场景必然失败

### 5.2 人工智能如何影响界面设计

- AI 用于搜索、素材生成、内容生成
- 设计师角色从「生产者」转向「策展人 + 编辑」

### 5.3 基于意图的交互设计

- 意图交互:用户表达「想要做什么」,系统理解并执行
- 与传统 GUI 对比:输入、响应、反馈、学习成本都不同
- 意图的设计和泛化是核心难题

### 5.4 基于人工智能的设计事项

- 合理收集数据 + 优化模型(避免偏差)
- 注意用户隐私(数据收集的合法性)

### 5.5 智能设计:基于「准确率」和「兜底」的设计 ⭐

#### 准确率阈值

```
99% → 直接展示
80-99% → 展示 + 提示确认
60-80% → 展示备选项
<60% → 兜底流程
```

#### 兜底 4 个层次(语音案例)

1. **告知型**:「抱歉,我听不懂」 —— 成本低,高频减分
2. **搜索型**:把语句传给搜索 —— 适合查询类,不适合指令
3. **闲聊型**:传给第三方闲聊机器人 —— 成本最高,效果最好
4. **Q&A 型**:传给产品内问答 —— 适合「怎么打开 X」

四种策略**互斥**,不能组合。

## 关键洞察

> 智能设计的设计过程是抽象的,可能不存在可视化过程,即使有也可能是「一步完成」,设计师很可能不知道自己该做什么。「准确率 + 兜底」是处理这种抽象性的唯一框架。
